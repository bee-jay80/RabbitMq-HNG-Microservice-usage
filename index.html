API Gateway

Ingress & Orchestration. Validates incoming requests, handles authentication, retrieves user data, and publishes the structured notification message to the queue.

Key Responsibilities:

  • Calls User Service (Sync).
  • Publishes message to RabbitMQ (Async).

Communication Flow:

Input: Synchronous REST requests from clients.

Output: JSON user profile (Sync); Message to RabbitMQ (Async).

RabbitMQ Contract & Flow

This panel details the specific broker targets. The shared setup uses the **notifications.direct** exchange for all publishing.

notifications.direct

Message Flow Diagram

Gateway ➡️ notifications.direct ➡️ Routing Key

|-- notify.email ➡️ email.queue ➡️ Email Service

|-- notify.push ➡️ push.queue ➡️ Push Service

|-- (Failure) ➡️ notifications.dlx ➡️ failed.queue

Code Implementation: Publisher Logic

The Gateway must implement this logic to publish the message using the correct exchange and routing keys defined in the contract.

Consumer Critical Logic:

Remember to implement **Idempotency Checks** and use **`basic_ack`** on success or **`basic_reject/nack`** (with requeue) on failure to ensure message integrity and utilize the DLQ for permanent failures.
